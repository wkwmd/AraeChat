# Gate: G-19 — Atomic write Option 3 (no contamination)

## SSOT anchors
- spec/ssot.yaml: atomic_write_option3.contamination_prevention
- spec/ssot.yaml: atomic_write_option3.tmp_naming
- spec/ssot.yaml: atomic_write_option3.os_calls.windows

## Goal
Windows에서 Option 3 원자 쓰기 규칙을 구현하여, 실패 시 out 오염이 없고 성공 시 원자 교체가 이루어짐을 증명한다.

## Checks (must pass)
- [ ] 실패 시 `out.bin`이 생성/변경되지 않는다.
- [ ] 성공 시 tmp 파일에 스트리밍 write 후 원자 rename/replace로 `out.bin`이 갱신된다.
- [ ] tmp naming:
  - [ ] `.<basename>.belowc.tmp` 사용
  - [ ] 충돌 시 `.<basename>.belowc.tmp.<n>` 최대 8회 시도
- [ ] OS 호출 규약:
  - [ ] CreateFileW(tmp)
  - [ ] rename preferred: SetFileInformationByHandle(FileRenameInfoEx)
  - [ ] fallback: MoveFileExW(REPLACE_EXISTING|WRITE_THROUGH)

## Test vectors
- 실패 케이스: 입력에 SyntaxError를 넣어 중간 실패 유도 → out.bin 오염 없음
- 성공 케이스: 정상 입력 → out.bin 생성/교체
- 충돌 케이스: tmp 기본 이름을 사전에 점유 → .tmp.1 등으로 회피
